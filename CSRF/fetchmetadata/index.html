Transfer Page Protected by Fetch Metadata

This page demonstrates server-side CSRF defense using Fetch Metadata headers. The transfer uses a plain POST form, but the server blocks suspicious cross-site requests.

How this is defended

  • Server validates Sec-Fetch-Site on unsafe methods.
  • Cross-site state-changing requests are rejected with HTTP 403.
  • Same-origin navigation requests can proceed, while suspicious contexts are blocked.

Typical defenses to combine with this: CSRF tokens, SameSite cookies, and Origin/Referer validation.
