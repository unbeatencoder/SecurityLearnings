Double Submit Cookie CSRF Demo

This page demonstrates the double submit cookie pattern with a static token. The same token is sent in a cookie and in this form field; the server compares both.

Why this is protected

  • State-changing action includes a CSRF token field.
  • Server also receives csrf_token cookie automatically.
  • Transfer is allowed only when form token equals cookie token.
  • This static-token version is demo-only; production should use unpredictable tokens.

Typical defenses include CSRF tokens, SameSite cookies, and Origin/Referer checks.
